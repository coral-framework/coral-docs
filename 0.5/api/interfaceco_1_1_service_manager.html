Detailed Description

Manages the system's services database, providing contextualized service lookups.

A module may publish its services with the ServiceManager in order to make them available for consumption by other modules.

Services are always identified by interface type. An interface instance may be published as either the global provider of a certain service (e.g. using addService()), or as a specialized service provider for clients of a certain interface type (e.g. using addServiceForType()). Whenever a service is published twice, the latest provided instance replaces the pre-existing instance.

The ServiceManager supports three kinds of lookups:

  • Lookup by service type: getService().
  • Lookup by service type + client type: getServiceForType().
  • Lookup by service type + client instance: getServiceForInstance().

void addServiceImplementation ( in co::InterfaceType  serviceType,
in string  componentName 
) throws co::IllegalArgumentException, co::NoSuchInterfaceException, co::MissingServiceException

Publishes a global provider of serviceType with lazy instantiation.

The service is immediately declared as available, but its instance is not created until the service is requested (unless the isLazy attribute is currently set to false).

To create the service instance, this manager will attempt to instantiate a component using the given componentName and get its facet of type serviceType. If an error occurs, the service will be removed and the calling method will raise a MissingServiceException.

Exceptions:
IllegalArgumentExceptionif serviceType is null or componentName does not identify a valid component type.
NoSuchInterfaceExceptionif the named component type does not provide an interface of type serviceType.
MissingServiceExceptionif isLazy is false and the service cannot be instantiated.

Publishes a provider of serviceType for clients of type clientType, with lazy instantiation.

The service is immediately declared as available, but its instance creation is delayed until the service is requested (unless the isLazy attribute is currently set to false).

To create the service instance, this manager will attempt to instantiate a component using the given componentName and get its facet of type serviceType. If an error occurs, the service will be removed and the calling method will raise a MissingServiceException.

Exceptions:
IllegalArgumentExceptionif a) serviceType or clientType are null; or b) componentName does not identify a valid component type; or c) clientType is the co.Interface interface (in this case you should call addService() instead).
NoSuchInterfaceExceptionif the named component type does not provide an interface of type serviceType.
MissingServiceExceptionif isLazy is false and the service cannot be instantiated.

co::Interface getServiceForInstance ( in co::InterfaceType  serviceType,
in co::Interface  clientInstance 
) throws co::IllegalArgumentException, co::MissingServiceException

Gets the most appropriate provider of serviceType for the given clientInstance.

This method first checks if the clientInstance's component provides an interface of type serviceType. If so, that interface is returned. Otherwise, this method behaves like a call to getServiceForType( service, clientInstance->getInterfaceType() ).

Exceptions:
IllegalArgumentExceptionif any of the arguments are null.
MissingServiceExceptionif no suitable instance of serviceType is available for clientInstance.
co::Interface getServiceForType ( in co::InterfaceType  serviceType,
in co::InterfaceType  clientType 
) throws co::IllegalArgumentException, co::MissingServiceException

Gets the most appropriate provider of serviceType for clients of the specified clientType.

If a service instance was not specifically added for the clientType, this method will pick the first service instance available for a super-type of clientType (by doing a depth-first search in the list of superInterfaces of clientType). Finally, if no specialized instance is available for clientType, this method will return the global provider of serviceType.

Exceptions:
IllegalArgumentExceptionif any of the arguments are null.
MissingServiceExceptionif no suitable instance of serviceType is available for clientType.
