Detailed Description

Allows sets of interdependent types to be created atomically.

Member Function Documentation

void commit (  ) throws co::MissingInputException, co::UnsupportedOperationException

Creates all types associated with this transaction, registering them with the system's TypeManager.

This validates any unvalidated type, which includes checking for missing data and semantic errors. If any type fails any of the checks, an exception is thrown and the user is expected to call rollback().

Exceptions:
MissingInputExceptionif any type is missing required input.
SemanticExceptionif one or more of the types contain semantic errors (e.g. cyclic inheritance).
UnsupportedOperationExceptionif a commit was already attempted, or if the transaction was rolled back.
void rollback (  )

Destroys and removes all references to partially created types associated with this transaction.

Exceptions:
UnsupportedOperationExceptionif the transaction is already dead (i.e. committed or rolled back).
