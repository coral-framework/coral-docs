Detailed Description

An abstract artifact (such as a dynamic library or a script bundle) loaded as part of a Module.

This interface is meant to control the life-cycle of such artifact.

Member Function Documentation

Disposal Phase - responsible for freeing resources allocated by the ModulePart.

void initialize ( in co::Module  module )

Initialization Phase - registration of services provided by the ModulePart.

Note:
This phase should install Reflectors for all module types. If a missing Reflector is detected after the initialization phase is complete, the module will be aborted.
