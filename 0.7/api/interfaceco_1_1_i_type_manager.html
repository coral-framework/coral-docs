Detailed Description

Manages the database of loaded types, providing methods to load and create new types.

Member Function Documentation

co::INamespace findNamespace ( in string  fullName)

Searches the namespace hierarchy for a namespace with the given fully-qualified name.

Returns null if the namespace does not exist (i.e. has not been created yet).

co::IType findType ( in string  fullName)

Searches the database of loaded types for a type with the given fully-qualified name.

A suffix "[]" may be added to the name of a type to search for its corresponding IArray. Returns null if a type with the given fullName is not currently loaded.

co::IArray getArrayOf ( in co::IType  elementType) throws co::IllegalArgumentException

Creates or retrieves an existing IArray whose elements are of type elementType.

Exceptions:
IllegalArgumentExceptionif elementType is null, an array, an exception or a component.
co::IType getType ( in string  typeName) throws co::IllegalArgumentException, co::TypeLoadException

Either retrieves or tries to load the type with the given fully-qualified typeName.

This can also retrieve or create arrays: if typeName ends with "[]", this method will call getArrayOf() passing as argument the element type specified in typeName.

Exceptions:
IllegalArgumentExceptionif the passed typeName is empty.
TypeLoadExceptionif any error occurs while loading the type.
IllegalArgumentExceptionwhen dealing with arrays, this is thrown if typeName specifies an invalid array element type (i.e. an array, exception or component).
co::IType loadType ( in string  typeName,
out co::CSLError[]  errorStack 
)

A lower-level counterpart of getType() that either retrieves or tries to load a type without raising exceptions.

In case of errors, this method will add detailed information to the errorStack and return null. The errorStack is sorted from the outermost to the innermost error.

Note:
This method does NOT create arrays, but can retrieve existing array types.
