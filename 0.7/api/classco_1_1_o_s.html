Searches for a file in a space defined by a combination of two string lists.

File path combinations are evaluated in the same order they're listed, and in case of ambiguities the search always returns the first match.

Parameters:
[in]listAList of base directory paths (e.g. pass the CORAL_PATH here).
[in]listBList of relative file paths (e.g. potential file names).
[out]pathPath to the first file found (only set if the method returns true).
[out]partAoptional; set with the substring of path that came from listA.
[out]partBoptional; set with the substring of path that came from listB.
Returns:
true if a file is found; false otherwise.

Searches for a file in a space defined by a combination of three string lists.

File path combinations are evaluated in the same order they're listed, and in case of ambiguities the search always returns the first match.

Parameters:
[in]listAOuter-most list of base directory paths (e.g. pass the CORAL_PATH here).
[in]listBCentral list of relative directory paths (e.g. pass a list of module names here).
[in]listCInner-most list of relative file paths (e.g. pass a list of file names here).
[out]pathPath to the first file found (only set if the method returns true).
[out]partAoptional; set with the substring of path that came from listA.
[out]partBoptional; set with the substring of path that came from listB.
[out]partCoptional; set with the substring of path that came from listC.

true if a file is found; false otherwise.
