Detailed Description

Controls the life-cycle of an abstract artifact (such as a dynamic library or a script bundle) loaded as part of a module.

Member Function Documentation

void initialize ( in co::IModule  module)

Initialization Phase - registration of services provided by the IModulePart.

Note:
This phase should install Reflectors for all module types. If a missing IReflector is detected after the initialization phase is complete, the module will be aborted.
void integrate ( in co::IModule  module) throws co::MissingServiceException

Integration Phase - consumption of services provided by other modules.

May throw an exception to indicate that the module cannot be loaded due to a missing service.

Exceptions:
MissingServiceExceptionif a required service was not available.
