Detailed Description

Basic type interface.

Member Data Documentation

readonly co::Uuid binarySignature

The type's binary signature: a version hash for checking if a type's binary interface has changed.

This can be used to check if two interfaces are binary compatible at the C++ ABI level.

readonly co::IReflector currentReflector

Similar to 'reflector', but getting this field never causes a reflector to be loaded.

This returns the currently set reflector, which can be null.

readonly string fullName

The type's fully qualified name.

If the type is in the global namespace, the full name is equivalent to the type's local name. Otherwise, the full name is equivalent to (type.namespace.fullName + "." + type.name).

readonly co::Uuid fullSignature

The type's full signature: a version hash for checking if a type has changed over time.

This considers changes to the type's binary interface, as well as other aspects that could affect source-code compatibility (e.g. struct field names, raised exceptions, etc.).

readonly string name

The type's local name (not considering its namespace).

For instance, for the type liquid.drinkable.Water, the local name would be Water.

co::IReflector reflector

The namespace where the type is defined.

The type's reflector.

Getting this field may cause the type's module to be loaded, in order to instantiate a reflector. Therefore, getting this field may raise a ModuleLoadException indicating that the reflector could not be obtained.
