Detailed Description

Manages the system's services database, providing contextualized service lookups.

A module may publish services with the IServiceManager in order to make them available for consumption by other modules.

Services are always identified by their IInterface type. A service may be published as either the global provider of a certain type of service (e.g. using addService()), or as a specialized service provider for clients of a certain interface type (e.g. using addServiceForType()). Whenever a service is published twice, the latest provided instance replaces the pre-existing one.

The service manager supports three kinds of lookups:

  • Lookup by service type: getService().
  • Lookup by service type + client type: getServiceForType().
  • Lookup by service type + client instance: getServiceForInstance().

void addServiceProvider ( in co::IInterface  serviceType,
in string  componentName 
) throws co::IllegalArgumentException, co::NoSuchPortException, co::MissingServiceException

Publishes a global provider of serviceType with lazy instantiation.

The service is immediately declared as available, but its instance is not created until the service is requested (unless the isLazy is currently set to false).

To create the service instance, this manager will attempt to instantiate a component using the given componentName and get its facet of type serviceType. If an error occurs, the service will be removed and the calling method will raise a MissingServiceException.

Exceptions:
IllegalArgumentExceptionif any arguments are null or componentName is not a valid component.
NoSuchPortExceptionif the component does not provide a facet of type serviceType.
MissingServiceExceptionif isLazy is false and the service cannot be instantiated.

Publishes a custom provider of serviceType for clients of type clientType, with lazy instantiation.

The service is immediately declared as available, but its instance creation is delayed until the service is requested (unless the isLazy is currently set to false).

To create the service instance, this manager will attempt to instantiate a component using the given componentName and get its facet of type serviceType. If an error occurs, the service will be removed and the calling method will raise a MissingServiceException.

co::IService getServiceForInstance ( in co::IInterface  serviceType,
in co::IService  client 
) throws co::IllegalArgumentException, co::MissingServiceException

Gets the most appropriate provider of serviceType for the given clientInstance.

This method first checks if the clientInstance's component provides an interface of type serviceType. If so, such interface is returned. Otherwise, this method behaves like a call to getServiceForType( service, client->getInterface() ).

Exceptions:
IllegalArgumentExceptionif any of the arguments are null.
MissingServiceExceptionif no suitable instance of serviceType could be found.
co::IService getServiceForType ( in co::IInterface  serviceType,
in co::IInterface  clientType 
) throws co::IllegalArgumentException, co::MissingServiceException

Gets the most appropriate provider of serviceType for clients of the specified clientType.

If a service was not specifically added for the clientType, this method will pick the first service available for a super-type of clientType. Finally, if no custom service is found for for clientType, this method will return the global provider of serviceType.

Exceptions:
IllegalArgumentExceptionif any of the arguments are null.
MissingServiceExceptionif no suitable instance of serviceType is available for clientType.
